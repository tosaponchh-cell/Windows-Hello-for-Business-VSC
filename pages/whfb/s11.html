PIN Reset Mechanism

Destructive Reset flow, Cloud PIN Reset Service และความต่างระหว่าง destructive vs non-destructive

PIN reset ใน WHfB เป็นกระบวนการ Destructive — key pair เดิมถูกทำลาย แล้วสร้างใหม่ทั้งหมด เพราะ PIN คือ local gesture ที่ protect private key ใน TPM ไม่มีกลไก "unlock PIN เก่าด้วย admin"

Destructive Reset Flow

1
ลืม PIN หรือ Lockout

พิมพ์ PIN ผิดเกิน threshold (default 5 ครั้ง) TPM จะ lockout ชั่วคราว (anti-hammering)

2
ทำลาย NGC Container

NGC container ที่เก็บ WHfB private key ถูก delete ออกจาก TPM โดยสมบูรณ์ — private key ไม่มีในโลกอีกต่อไป

3
MFA Re-verification บังคับ

Windows ขอ MFA (Authenticator, TOTP, SMS) ผ่าน Entra ID — ป้องกันคนอื่นทำ PIN reset แทน

4
Provisioning ใหม่ทั้งหมด

TPM สร้าง key pair ใหม่ → register public key ใหม่ → เขียนทับ msDS-KeyCredentialLink

5
ตั้ง PIN ใหม่ + PRT ใหม่

provisioning สำเร็จ → PRT ใหม่ออกให้ → Cloud SSO กลับมาทำงาน

เปิด Cloud PIN Reset (Self-Service)

// Intune — PassportForWork CSP
./Device/Vendor/MSFT/PassportForWork/{TenantID}/EnablePinRecovery = True

// GPO
Computer Config → Windows Components → Windows Hello for Business
  → Enable PIN Recovery = Enabled
// เมื่อเปิด: สร้าง recovery secret ใน TPM
// → reset PIN โดยไม่ต้อง full destructive reset
// (key pair เดิมยังอยู่ แค่เปลี่ยน PIN gesture)
