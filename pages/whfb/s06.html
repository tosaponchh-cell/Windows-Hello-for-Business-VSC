Cloud Kerberos Trust — PKINIT Deep Dive

PA-PK-AS-REQ structure, paChecksum binding และ Partial TGT → Full TGT exchange

6.1 PA-PK-AS-REQ — ข้อมูลที่ Hash และ Sign

AuthPack {                       // ← DER encode แล้ว sign ทั้ง struct
    pkAuthenticator: {
        cusec:      <microseconds>,
        ctime:      <KerberosTime UTC>,
        nonce:      <INT32 random>,
        paChecksum: SHA256( DER_Encode(KDC-REQ-BODY) )
                   ↑ bind signature เข้ากับ request body ทั้งหมด
                     ป้องกัน signature substitution attack
    },
    supportedCMSTypes: [ecdsa-with-SHA256, sha256WithRSAEncryption]
}
// CMS SignedData wrapping
PA-PK-AS-REQ.signedAuthPack = SignedData {
    digest:    SHA256( DER_Encode(AuthPack) ),
    signature: ECDSA_Sign(WHfB_privateKey, digest),
    sid:       subjectKeyIdentifier = KeyID
}
💡 paChecksum คืออะไร

paChecksum = SHA256(KDC-REQ-BODY) bind signature เข้ากับ request body ทั้งหมด ป้องกัน attacker ที่ดัก PKINIT packet แล้วเอา signature ไปใส่ใน request อื่น (signature substitution attack)

6.2 Partial TGT → Full TGT Exchange

Step 1: Client → Entra ID : PA-PK-AS-REQ (signed with WHfB key)
        Entra ID → Client  : AS-REP + Partial TGT {
            crealm: "KERBEROS.MICROSOFTONLINE.COM",
            ticket: <encrypted with AzureADKerberos krbtgt key>
        }

Step 2: Client → on-prem DC : TGS-REQ {
            padata:   [Partial TGT],
            req-body: { sname: krbtgt/CORP.LOCAL }
        }

// DC ตรวจ Partial TGT ผ่าน AzureADKerberos object:
AzureADKerberos:
  CN=AzureADKerberos, OU=Domain Controllers, DC=corp, DC=local
  → krbtgt_AzureAD key สำหรับ decrypt Partial TGT
