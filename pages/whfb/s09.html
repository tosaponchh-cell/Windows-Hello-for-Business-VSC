Troubleshooting & Event Logs

Event IDs หลัก, Diagnostic Commands และ Error Codes ที่พบบ่อย

Event Logs ที่ต้องดู

Event 1001WHfB Provisioning OK — HelloForBusiness/Operational
KeyID: {SHA256 hash of public key} KeySource: 0=AD | 1=AzureAD
Event 1002WHfB Provisioning FAILED — HelloForBusiness/Operational
ErrorCode: 0x801C03ED Cause: MFA ไม่ผ่าน, CA policy ปิดกั้น, device ยังไม่ registered
Event 4768Kerberos TGT — Security Log (บน DC)
PreAuthType: 16 = PKINIT ← WHfB ทำงาน (ไม่ใช่ password) Result Code: 0x0 = Success
Event 5061Cryptographic Operation — System Log
Algorithm: ECDSA_P256 Result: 0x0 ← ทุก sign operation log ที่นี่

Diagnostic Commands

dsregcmd /status
// AzureAdJoined:YES + DomainJoined:YES = Hybrid OK
// NgcSet:YES = WHfB provisioned | NgcKeyId:{GUID}
// AzureAdPrt:YES = PRT มีอยู่

klist            // TGT ปัจจุบัน
klist purge      // ล้าง ticket cache

Get-WinEvent -LogName "Microsoft-Windows-HelloForBusiness/Operational" | Select-Object -First 20

Get-ADUser -Identity username -Properties "msDS-KeyCredentialLink"

certutil -deleteHelloContainer  // บังคับ re-provision
certutil -scinfo                 // WHfB cert info (Certificate Trust)

Error Codes

Error Codeความหมายแนวทาง
0x801C03EDKey registration failedMFA ไม่ผ่าน หรือ CA policy ปิดกั้น
0x801C044DTPM attestation failedTPM firmware เก่า หรือ attestation policy strict
0x80090016NTE_BAD_KEYSETContainer เสียหาย — ใช้ certutil -deleteHelloContainer
0xC000006DSTATUS_LOGON_FAILUREKey Trust: msDS-KeyCredentialLink ยังไม่ sync
0x801C0451Cloud TGT errorAzureADKerberos object ยังไม่ถูกสร้าง
0x801C044CNo TPMVM ไม่ได้ pass-through TPM หรือ TPM 1.2
4768 result 0x1APKINIT not supportedDC version เก่า หรือ NTAuth store ไม่มี cert
✅ Hybrid Checklist

1. dsregcmd → AzureAdJoined + DomainJoined = YES
2. MFA สำเร็จก่อน provisioning
3. Key Trust: ตรวจ Entra Connect sync + msDS-KeyCredentialLink
4. Cert Trust: ตรวจ CA reachability, NTAuth store, CRL/OCSP
5. Cloud Kerberos: ตรวจ AzureADKerberos object + DC ≥ 2016
