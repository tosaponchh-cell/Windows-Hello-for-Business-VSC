04 — CRYPTO INTERNALS

Certificate Trust — X.509, CSR & Chain Validation

ข้อมูลที่ถูก hash ในแต่ละขั้นตอน ตั้งแต่ CSR จนถึง CA signing และ chain validation

4.1 CSR — ข้อมูลที่ถูก Hash แล้วเซ็น

Step 1: tbsBytes = DER_Encode( CertificationRequestInfo {
    version:      0,
    subject:      CN=user@domain.com,
    subjectPKInfo: { algorithm: id-ecPublicKey, publicKey: <WHfB_pubkey> },
    attributes: extensionRequest {
        subjectAltName: UPN: user@domain.com,
        keyUsage:       digitalSignature,
        extKeyUsage:    [smartcardLogon, clientAuth],
        certTemplate:   <WHfB template name>
    }
})
Step 2: digest    = SHA256(tbsBytes)
Step 3: signature = ECDSA_Sign(WHfB_privateKey, digest)

Step 1: tbsCertBytes = DER_Encode(TBSCertificate)
Step 2: certDigest   = SHA256(tbsCertBytes)
Step 3: caSignature  = RSA_PKCS1_Sign(CA_privateKey, certDigest)

4.3 Chain Validation — 6 ขั้นตอน

1
Signature Verification

ดึง issuer cert → SHA256(TBSCertificate) → verify ด้วย issuer's public key

2
Validity Period

ตรวจ notBefore ≤ now ≤ notAfter ทุก cert ในสาย รวมถึง CA cert

3
Revocation (CRL / OCSP)

ดึง CRL จาก CDP extension หรือ query OCSP จาก AIA → ตรวจ serial number

4
NTAuth Store

Root CA ต้องอยู่ใน CN=NTAuthCertificates ใน AD — ถ้าไม่มี DC reject

5
EKU Validation

ต้องมี Smartcard Logon OID 1.3.6.1.4.1.311.20.2.2

6
Subject Mapping

DC map cert → AD user โดยใช้ UPN ใน SAN → userPrincipalName attribute
