01 — FOUNDATION

ภาพรวม & หลักการพื้นฐาน

WHfB แทน password ด้วย asymmetric key pair ที่ผูกกับ TPM hardware โดยตรง — ไม่มี shared secret ส่งผ่านเครือข่าย

Windows Hello for Business ไม่ใช่แค่ "MFA เพิ่มชั้น" แต่เป็นการออกแบบใหม่ทั้งหมด ไม่มี password ใน AD ที่ใช้ตรวจสอบ มีแค่ key pair ที่ผูกกับ hardware ของเครื่องนั้นเครื่องเดียว

หลักการ 3 ข้อ

1
Something you have — TPM

Private key ถูกสร้างและเก็บใน TPM chip ไม่สามารถ export ออกมาได้ แม้แต่ OS ที่ถูก compromise ก็เข้าถึงไม่ได้

2
Something you know/are — Gesture

PIN หรือ Biometric ไม่ได้ "ส่ง" ไปไหน แค่ "unlock" การใช้ private key ภายใน TPM ของเครื่องนั้นเท่านั้น

3
Identity binding — Public key ใน IdP

Entra ID / AD เก็บ public key คู่กัน การพิสูจน์ตัวตน = เซ็น challenge ด้วย private key ให้ IdP verify ด้วย public key

📌 PIN ≠ Password

PIN ไม่เคยออกจากเครื่อง ไม่ถูก hash เพื่อส่งเครือข่าย ไม่มีใน AD ถ้าขโมย PIN แต่ไม่มีเครื่อง → ใช้งานไม่ได้
