PRT & Cloud SSO

Primary Refresh Token คือหัวใจของ Cloud SSO — ผูกกับ TPM session key ทำให้ขโมยไม่ได้

💡 PRT คืออะไร

PRT เป็น long-lived token (อายุ 14 วัน ต่ออายุได้ถึง 90 วัน) ที่ Entra ID ออกให้กับ user + device คู่นั้น ใช้แลกเป็น Access Token สำหรับ M365, SharePoint, Teams ได้ทั้งหมดผ่าน Web Account Manager (WAM)

PRT Request Flow ขณะ WHfB Login

1
TPM Unlock

WHfB private key พร้อม — CloudAP plugin ใน Windows ถูกเรียกพร้อมกัน

2
Request ไป DRS

ส่งไปที่ Device Registration Service (enterpriseregistration.windows.net) พร้อม User identity + Device identity + Nonce

3
WHfB Private Key เซ็น Nonce

TPM เซ็น nonce → Entra ID verify ด้วย registered public key

4
DRS ออก PRT + Session Key

Entra ID ออก PRT (JWT) + session_key (AES-256) — session_key ถูก encrypt ด้วย transport key ของ device → TPM เท่านั้น decrypt ได้

5
TPM Decrypt Session Key

session_key ใช้งานได้ใน memory แต่ไม่เคย expose เป็น plaintext

6
PRT เก็บใน LSASS Cache

WAM ดูแลการแลก PRT เป็น Access Token ทุก app อัตโนมัติ

PRT Cookie — ทำไมขโมย PRT แล้วก็ยังใช้ไม่ได้

// PRT JWT payload claims
{ "amr": ["ngcmfa"], "device_id": "{GUID}", "exp": <+14 days> }

// ทุกครั้งที่แลก Access Token ต้องแนบ PRT Cookie
PRT Cookie = Base64Url({
    "request_nonce": <nonce จาก Entra — ป้องกัน replay>,
    "iat":           <timestamp>
}, signed_with = HMAC_SHA256(session_key, header||payload))
// session_key อยู่ใน TPM-protected memory เท่านั้น
// ขโมย PRT แต่ไม่มีเครื่อง → ทำ PRT cookie ไม่ได้

amr Claims — WHfB = MFA ในสายตา Entra

amr valueความหมายMFA level
ngcmfaWindows Hello for Business✓ MFA satisfied
pwdPassword only✗ ต้อง step-up MFA
mfaPassword + Authenticator app✓ MFA satisfied
fidoFIDO2 security key✓ MFA satisfied
