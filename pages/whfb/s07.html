07 — OPERATIONS

Provisioning Flow — Hybrid AAD

ขั้นตอนตั้งแต่ login ครั้งแรกจนได้ WHfB credential — ทุก trust model ใช้ core flow เดียวกัน

1
Trigger: First Sign-In + Policy

WHfB provisioning เปิดหลัง user sign-in เมื่อ device เป็น Hybrid AAD Joined และมี WHfB policy (GPO หรือ Intune MDM)

2
MFA Verification (บังคับ)

Windows ขอ MFA ผ่าน Entra ID Conditional Access ก่อนสร้าง key ทุกครั้ง

3
TPM Key Generation

TPM2_Create สร้าง ECC P-256 key pair ภายใน TPM ภายใต้ Storage Root Key (SRK) — private key ไม่ออกจาก TPM เลย

4
Attestation

TPM2_CertifyCreation สร้าง attestation statement → ส่งพร้อม public key ให้ Entra ID ตรวจ firmware + manufacturer cert chain

5
Public Key Registration

Cloud Kerberos / Key Trust: public key ลงทะเบียนกับ Entra ID → sync ลง AD → เขียน msDS-KeyCredentialLink
Certificate Trust: CSR ส่งไป CA via NDES → cert เก็บใน user store

6
PIN / Biometric Setup

User ตั้ง PIN (6+ digits) หรือ fingerprint/face — เป็น local gesture ที่ unlock private key ใน TPM

7
Verification & Complete

Windows ทดสอบ credential ด้วย sign operation จริง → ถ้าผ่าน log Event 1001 → provisioning สำเร็จ
