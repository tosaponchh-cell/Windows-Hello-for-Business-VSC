TPM Deep Dive

ข้อมูลอะไรบ้างถูก hash — ตั้งแต่ Key Generation จนถึง Attestation Statement

3.1 Key Name = Hash ของ publicArea

ใน TPM 2.0 ทุก object มี Name ซึ่งเป็น Hash(publicArea) ไม่ใช่แค่ GUID — ชื่อนี้ใช้ binding attestation เข้ากับ key จริง

// TPM2_Create — สร้าง WHfB key
TPMT_PUBLIC publicArea = {
    type:         TPM_ALG_ECC,
    nameAlg:      TPM_ALG_SHA256,
    objectAttributes: {
        fixedTPM:     1,  // key ออกจาก TPM ไม่ได้
        userWithAuth: 1,  // ต้อง PIN gesture ก่อนใช้
        sign:         1,  // sign ได้ / decrypt ไม่ได้
    },
    parameters: { curveID: TPM_ECC_NIST_P256 }
}
Key_Name = 0x000B || SHA256( DER_Serialize(publicArea) )
           ↑ 0x000B = TPM_ALG_SHA256 prefix (2 bytes)

3.2 TPMS_ATTEST — โครงสร้างที่ TPM เซ็น

TPMS_ATTEST (TCG Spec Part 2, §10.12.8)
+0
magic
UINT32
0xff544347 ("ÿTCG") — ป้องกัน signature confusion
+4
type
TPMI_ST_ATTEST
TPM_ST_ATTEST_CREATION (0x8017)
+6
qualifiedSigner
TPM2B_NAME
Name ของ Attestation Key = SHA256(AK_publicArea)
+var
extraData
TPM2B_DATA
Nonce จาก Entra ID — ป้องกัน replay
+var
firmwareVersion
UINT64
TPM firmware version (Entra ตรวจสอบ)
+8
attested.objectName
TPM2B_NAME
Key_Name ของ WHfB key = 0x000B||SHA256(publicArea)
+var
attested.creationHash
TPM2B_DIGEST
SHA256 ของ TPMS_CREATION_DATA
// ลำดับ hash และ sign จริง
Step 1: creationHash = SHA256( TPMS_CREATION_DATA {
    pcrDigest:  SHA256( PCR[0]||PCR[1]||...||PCR[n] ),
    parentName: SHA256( SRK_publicArea )
})
Step 2: sig = ECDSA_Sign( AK_key, SHA256( marshal(TPMS_ATTEST) ) )
Step 3: ส่ง { TPMS_ATTEST, sig, aikCert, WHfB_publicKey } → Entra ID
💡 Key Name ทำให้ attestation น่าเชื่อถือ

เวลา TPM ออก attestation มันรวม objectName = SHA256(publicArea) ไว้ด้วย ทำให้ Entra ID มั่นใจว่า public key ที่ลงทะเบียนตรงกับ key ใน TPM จริง ๆ
