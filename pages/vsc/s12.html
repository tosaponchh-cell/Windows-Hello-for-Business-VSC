12 — OPERATIONS

tpmvscmgr Deep Dive

รายละเอียดทุก parameter ของ tpmvscmgr.exe — /pin /adminkey /puk /attestation และ PIN Policy

⚠ บังคับ — Run as Administrator ทุกครั้ง

ถ้าไม่ elevate → Error: Access is denied (0x80070005) ทันที — Right-click CMD → Run as administrator

Full Command (Production)

tpmvscmgr.exe create /name "Client01" \
  /pin prompt \
  /pinpolicy minlen 10 uppercase REQUIRED lowercase REQUIRED digits REQUIRED specialchars REQUIRED \
  /adminkey prompt \
  /puk prompt \
  /attestation AIK_ONLY \
  /generate

/name — ชื่อ Container

Parameterความหมายหมายเหตุ
/name "ชื่อ"ชื่อของ VSC containerเป็นแค่ label — ระบบใช้เลข 0000, 0001... จริงๆ ในการ manage

/pin — PIN หลักสำหรับ Login

Optionผลลัพธ์Production?
/pin defaultPIN ถูกตั้งเป็น 12345678Lab เท่านั้น
/pin promptระบบถามให้ตั้ง PIN เอง (min 8 ตัว)✓ แนะนำ

/pinpolicy — กฎการตั้ง PIN

// ตัวอย่าง production policy
/pinpolicy minlen 10 uppercase REQUIRED lowercase REQUIRED digits REQUIRED specialchars REQUIRED

// options ทั้งหมด
minlen [N]          // ความยาวขั้นต่ำ (ต่ำสุด 4, default 8, production แนะนำ 10+)
uppercase  REQUIRED | ALLOWED | DISALLOWED
lowercase  REQUIRED | ALLOWED | DISALLOWED
digits     REQUIRED | ALLOWED | DISALLOWED
specialchars REQUIRED | ALLOWED | DISALLOWED  // !@#$%^&* ฯลฯ

/adminkey — กุญแจบริหาร (Admin Key)

Optionผลลัพธ์Production?
/adminkey default010203040506070801020304050607080102030405060708Lab เท่านั้น
/adminkey promptตั้งเอง — ต้องเป็น Hex 48 ตัวพอดี✓ แนะนำ
/adminkey randomสุ่ม — หายทันที ไม่มีทางกู้✗ ห้ามใช้ production
⚠ Format ของ adminkey prompt

ต้องเป็น ตัวเลข 0-9 และ A-F หรือ a-f เท่านั้น ความยาวรวม 48 ตัวพอดี
นอกเหนือจากนี้ → Error 0x80070057
ตัวอย่างที่ถูกต้อง: AABBCCDDEEFF00112233445566778899AABBCCDDEEFF0011

/puk — PIN Unlock Key (กุญแจกู้ PIN)

Optionผลลัพธ์Production?
/puk defaultPUK ถูกตั้งเป็น 123456789012Lab เท่านั้น
/puk promptตั้งเอง — เก็บไว้ใน Vault✓ แนะนำ
/puk randomสุ่ม — หายทันที กู้ PIN ไม่ได้ตลอดกาล✗ ห้ามใช้
📌 PUK คืออะไร

PUK (PIN Unlock Key) ใช้กู้ PIN เมื่อ user ใส่ผิดหลายครั้งจนขึ้น "The smart card is blocked" — IT Support ใช้ PUK นี้เพื่อ unlock โดยไม่ต้อง destroy VSC ใหม่ทั้งหมด ต้องเก็บ PUK ใน Password Vault เสมอ

/attestation — การยืนยัน Hardware Binding

Attestation คือกระบวนการพิสูจน์ว่า private key ถูกสร้างและผูกอยู่ใน TPM hardware จริง ไม่ใช่ software จำลอง

Modeทำงานอย่างไรต้องออก Internet?ต้องติดตั้งเพิ่ม?
AIK_ONLYสร้าง AIK ภายในเครื่อง TPM รับประกันตัวเอง❌❌
AIK_AND_CERT + Microsoft HASส่ง AIK ไป Microsoft Health Attestation Service บน Cloud✅❌
AIK_AND_CERT + HRA LANส่งผ่าน HRA Server ใน LAN → Issuing CA ออก AIK Cert❌✅ HRA Server

AIK_ONLY Flow (แนะนำสำหรับระบบปิด)

tpmvscmgr /attestation AIK_ONLY
    │
    ▼
TPM ในเครื่อง CLIENT01
    ├── สร้าง AIK (Attestation Identity Key)
    ├── บันทึก AIK ลงใน TPM
    └── จบกระบวนการ — ไม่มีการติดต่อภายนอกใดๆ
// Issuing CA ไม่เกี่ยวกับ Attestation เลย — เกี่ยวแค่ Certificate Enrollment ตามปกติ

AIK_AND_CERT + HRA Server Flow (ถ้าต้องการ Hardware-verified แบบ Local)

CLIENT01
    │  ส่ง AIK + EK (Endorsement Key จากโรงงาน)
    ▼
HRA Server  ← Windows Server role ใน LAN
    │  ขอ AIK Certificate
    ▼
Issuing CA
    ├── ตรวจสอบ EK ว่ามาจาก TPM ที่เชื่อถือได้
    ├── ✓ ผ่าน → ออก AIK Certificate
    └── ✗ ไม่ผ่าน → ปฏิเสธ
    │  ส่ง AIK Certificate กลับผ่าน HRA
    ▼
CLIENT01  ← VSC ถูก Attest แบบ Hardware-bound โดย CA ของคุณเอง
✅ แนะนำสำหรับโรงพยาบาล (Closed Network)

ใช้ AIK_ONLY — ลดความยุ่งยาก ไม่ต้องออก internet ไม่ต้องติดตั้งเพิ่ม และยังได้ hardware binding จาก TPM อยู่ เสริมด้วย GPO ผูก VSC กับเครื่อง (Machine-bound) แทน

Destroy Command

// ตรวจ instance ID ก่อน
certutil -scinfo   // ดูชื่อ instance เช่น ROOT\SMARTCARDREADER\0000

// ลบ VSC
tpmvscmgr.exe destroy /instance ROOT\SMARTCARDREADER\0000
// ⚠ ต้อง destroy ก่อนล้างเครื่อง (Clean Install) เสมอ เพื่อเคลียร์ TPM
