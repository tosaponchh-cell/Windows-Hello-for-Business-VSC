11 — OPERATIONS

Tips, Warnings & Enterprise Path

สิ่งที่ห้ามทำ, upgrade path สู่ production และ relationship กับ WHfB

สิ่งที่ต้องห้ามทำ (Never Do)

❌ ห้ามทำเหตุผล
ใช้ tpmvscmgr.msc แทน tpmvscmgr.exe.msc คือ Management Console snap-in ไม่ใช่ runnable command
รัน tpmvscmgr.exe โดยไม่ Run as administratorCommand จะ fail หรือ return Access Denied (0x80070005)
Enroll cert ขณะ login เป็น Administratorcertmgr.msc ต้องเปิดเป็น LabUser01 — cert ต้องอยู่ใน store ของ user นั้น
Skip Security tab ตอนสร้าง templateไม่มี Read + Enroll permissions = template invisible กับ user ทุกคน
Publish แค่ .crt โดยไม่มี .crl.crl จำเป็นสำหรับ revocation checking — ถ้าไม่มี Smart Card login ล้มเหลวทั้งหมด
ใช้ -setreg ca\CRLFlags +CRLF_REVCHECK_IGNORE_OFFLINEDisable core security control — ทำลายจุดประสงค์ทั้งหมดของ PKI
ทิ้ง ROOTCA ไว้แบบ powered onRoot CA private key exposure = compromise PKI trust chain ทั้งหมด

Enterprise Upgrade Path

Upgradeประโยชน์
IIS Web Server (HTTP CDP)Host CRL via HTTP — เร็วกว่า, accessible to non-domain devices
OCSP ResponderReal-time revocation check — ไม่ต้อง download CRL ทั้ง file
Secondary Issuing CAHigh availability — ถ้า ISSUINGCA fail, CA ตัวที่สองรับต่อ
Secondary DC (Replica)ถ้า DC01 fail, Kerberos และ CRL distribution fail ทั้งหมด
Group Policy Autoenrollmentแจกจ่าย certificates อัตโนมัติ ไม่ต้องให้ user รัน certmgr.msc
Windows Hello for BusinessMicrosoft's recommended replacement — TPM-backed, cloud-aware

VSC vs WHfB — ความสัมพันธ์

📌 Microsoft's Position (2019+)

Microsoft แนะนำ Windows Hello for Business หรือ FIDO2 keys สำหรับ new deployments Virtual Smart Cards ผ่าน tpmvscmgr.exe ยังได้รับการ support เต็มที่แต่ถือเป็น legacy path ตั้งแต่ Windows 10/11 และ Server 2019+
