Interpreting certutil -scinfo Output

Outputความหมาย
dwErrorStatus=0 ทุก CertContextAll clear — chain and revocation valid
CERT_TRUST_IS_OFFLINE_REVOCATION (1000040)CRL unreachable — publish และ import CRL ของ CA นั้น
CERT_TRUST_REVOCATION_STATUS_UNKNOWN (40)CRL ไม่พบที่ไหนเลย
CertContext[0][0] failsปัญหาที่ user certificate — ตรวจ enrollment และ UPN
CertContext[0][1] failsปัญหาที่ Issuing CA cert — Root CA CRL missing
Private key verifies OKVSC hardware healthy — card ไม่ใช่ปัญหา

CRL Resolution — 4 Phases

Phase A — ROOTCA: Refresh Root CA CRL
// certsrv.msc → Revoked Certificates → All Tasks → Publish → New CRL
// Copy LAB-RootCA.crl ไปที่ ISSUINGCA, DC01, CLIENT01
// Shut down ROOTCA ทันที

Phase B — ISSUINGCA: Update
// certlm.msc → Trusted Root → CRLs → Delete expired
certutil -addstore -f root "C:\Certs\LAB-RootCA.crl"
certutil -addstore -f CA   "C:\Certs\LAB-RootCA.crl"
certutil -dspublish -f "C:\Certs\LAB-RootCA.crl" RootCA
// Restart ISSUINGCA VM แล้ว publish Issuing CRL ใหม่
// certsrv.msc → Revoked Certificates → Publish

Phase C — DC01 (KDC): Update
certutil -addstore -f root "C:\Certs\LAB-RootCA.crl"
certutil -urlcache crl delete
certutil -setreg chain\ChainCacheResyncFiletime @now
// Restart DC01 เพื่อ flush KDC Kerberos cache

Phase D — CLIENT01: Update
certutil -addstore -f root "C:\Certs\LAB-RootCA.crl"
certutil -urlcache crl delete
// Restart CLIENT01 → ทดสอบ Smart Card login ด้วย PIN
