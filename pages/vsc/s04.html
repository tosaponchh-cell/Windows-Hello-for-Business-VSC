Section 3 — Sign .req บน ROOTCA (.req → .cer)

📌 INFO

ทำบน ROOTCA — เปิดเครื่องเฉพาะ section นี้ Shut down ทันทีหลัง export .cer

1
Copy .req ไปที่ ROOTCA

Power on ROOTCA → Copy ISSUINGCA.req จาก ISSUINGCA ไปที่ ROOTCA Desktop หรือ C:\Certs\

2
Submit .req ไปที่ Root CA

Open certsrv.msc บน ROOTCA → Right-click CA name → All Tasks → Submit new request... → เลือก .req file

3
Issue the Certificate

Click Pending Requests → Right-click request → All Tasks → Issue
Request ย้ายไป Issued Certificates = approved

4
Export .cer File

Click Issued Certificates → Double-click cert → Details → Copy to File...
Format: Base-64 encoded X.509 (.CER) — ห้ามเลือก DER
Save as: IssuingCA_Signed.cer

5
Copy .cer กลับ ISSUINGCA แล้ว Shut Down ROOTCA

Copy IssuingCA_Signed.cer ไปที่ ISSUINGCA (C:\Certs\) แล้ว shut down ROOTCA ทันที

⚠ Export Format สำคัญมาก

ต้องเลือก Base-64 encoded X.509 (.CER) เท่านั้น — ถ้าเลือก DER แล้ว ISSUINGCA จะไม่สามารถ install ได้ในขั้นตอนต่อไป

Section 4 — Install .cer บน ISSUINGCA & Start Service

📌 INFO

ทำบน ISSUINGCA — หลัง section นี้ CA service ควรแสดงสถานะ running (green)

1
Install CA Certificate

certsrv.msc บน ISSUINGCA → Right-click CA name → All Tasks → Install CA Certificate...
เลือก IssuingCA_Signed.cer จาก C:\Certs\

2
Start the Service

Right-click CA name → All Tasks → Start Service
ไอคอนควรเปลี่ยนเป็นสีเขียวแสดงว่า CA กำลัง running

3
Publish Issuing CA CRL

หลัง service start: Right-click Revoked Certificates → All Tasks → Publish → New CRL
ทำให้ CLIENT01 และ DC01 สามารถ download CRL ของ Issuing CA ได้

Health Check ก่อน Continue

สิ่งที่ตรวจExpected
certsrv.msc iconRunning / no red X
Event Viewer → Application logNo errors from source: CertSvc
certutil -pingCertUtil: -ping command completed successfully
⚠ CA Service Starts Then Stops

ถ้า CA service start แล้วหยุดทันที: มี expired CRL อยู่ใน local store
Fix: certlm.msc → ลบ expired CRL จาก Trusted Root → Certificate Revocation Lists → restart VM (ไม่ใช่แค่ restart service)
