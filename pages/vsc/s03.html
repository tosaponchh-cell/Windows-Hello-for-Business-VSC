03 — PKI BUILD

Build Issuing CA

ทำบน ISSUINGCA — ต้อง join domain, log in as Enterprise Admin

📌 INFO

ทำบน ISSUINGCA — ต้อง domain-joined, log in as lab\Administrator (Enterprise Admin)

Phase 1: Publish Root CA Files to Active Directory

ก่อนติดตั้ง role ต้อง publish Root CA files ลง AD ก่อน

1
Copy Root CA Files ไปที่ ISSUINGCA

วาง LAB-RootCA.crt และ LAB-RootCA.crl ใน C:\Certs\

2
Publish Root CA Certificate ลง AD 
certutil -dspublish -f "C:\Certs\LAB-RootCA.crt" RootCA
// Expected: CertUtil: -dsPublish command completed successfully
3
Publish Root CA CRL ลง AD 
certutil -dspublish -f "C:\Certs\LAB-RootCA.crl" RootCA
// Expected: Base CRL added to DS store. CertUtil: -dsPublish command completed successfully
4
Import ลง Local Machine Store 
certutil -addstore -f root "C:\Certs\LAB-RootCA.crt"
certutil -addstore -f root "C:\Certs\LAB-RootCA.crl"
certutil -addstore -f CA   "C:\Certs\LAB-RootCA.crl"
⚠ ERROR_DS_REFERRAL

ถ้าเจอ error นี้บน certutil -dspublish: Root CA เป็น Standalone ดังนั้น CRL มี 'UnavailableConfigDN' — แก้ไขโดยต้องใส่ RootCA parameter ต่อท้ายคำสั่งเสมอ

Phase 2: Install AD CS Role & Generate .req

5
Install Role

Server Manager → Add Roles → Active Directory Certificate Services → Certification Authority only → Install

6
Configure: Enterprise Subordinate CA

Click yellow flag → Configure AD CS
Setup Type: Enterprise CA (available เพราะ join domain แล้ว)
CA Type: Subordinate CA | RSA 2048, SHA-256
CA Name: LAB-IssuingCA
⚠ หน้าสำคัญ — Certificate Request: เลือก Save a certificate request to file
จด path ของ .req file ก่อน click Configure เช่น C:\ISSUINGCA.req

💡 TIP

CA service จะแสดงสถานะ stopped หลัง Configure — เป็นเรื่องปกติ มันไม่สามารถ start ได้จนกว่าจะติดตั้ง signed .cer ใน Section 4
