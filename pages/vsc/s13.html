IT Support Flow

ขั้นตอนการ Setup PC, Certificate Enrollment และ workflow แก้ปัญหา PIN/PUK/TPM Lockout

0. Active Directory — GPO ที่ต้องตั้งก่อน

GPOPathเหตุผล
Unblock smart card screen Computer Config → Admin Templates → Windows Components → Smart Card → Allow Integrated Unblock screen to be displayed at the time of logon ให้ UI แสดงช่อง PUK ที่ login screen เมื่อ card blocked
Remove Change Password User Config → Admin Templates → System → Ctrl+Alt+Del Options → Remove Change Password ป้องกัน user เปลี่ยน PIN เอง นอกช่องทาง IT

1. Setup PC (ก่อน Deploy)

1
ติดตั้ง Windows

Clean install, update ให้เป็น version ล่าสุด

2
สร้าง Local Admin

สร้าง local user: lcadmin สำหรับใช้ maintain เครื่อง

3
เชื่อมต่อ VLAN

เชื่อม VLAN ที่ถูกต้อง (เช่น VLAN 105) ก่อน join domain

Join เข้า domain แล้ว restart

5
ตรวจสอบ TPM

Win + R → tpm.msc
Ready = เปิดแล้ว ✓ | Disabled = ต้องเปิดใน BIOS

2. ติดตั้ง VSC

📌 ต้อง Login เป็น Domain User ก่อน

Login ด้วย User domain ที่จะใช้งานจริง ก่อนรัน tpmvscmgr.exe

// เปิด CMD as Administrator แล้วรัน:
tpmvscmgr.exe create /name "Client01" \
  /pin prompt \
  /pinpolicy minlen 10 uppercase REQUIRED lowercase REQUIRED digits REQUIRED specialchars REQUIRED \
  /adminkey prompt \
  /puk prompt \
  /attestation AIK_ONLY \
  /generate
⚠ บันทึก PIN และ PUK ลง Vault ทันที

adminkey และ PUK ต้องบันทึกลง Password Vault ก่อนปิดหน้าต่าง CMD — ถ้าหายไม่มีทางกู้ได้นอกจาก Destroy + Create ใหม่

3. Certificate Enrollment

1
Login ด้วย Domain User

ต้องเป็น user ที่จะใช้งานจริง ไม่ใช่ local admin

2
เปิด certmgr.msc

Win + R → certmgr.msc → Enter

3
Request New Certificate

Personal → คลิกขวา Certificates → All Tasks → Request New Certificate...
Next → Active Directory Enrollment Policy → Next → เลือก Smartcard template → Enroll

4
ใส่ PIN เมื่อถูกถาม

ใส่ PIN ที่ตั้งไว้ตอน tpmvscmgr → Status: Succeeded → Finish

5
ตรวจสอบ

certmgr.msc → Personal → Certificates → ต้องเห็น cert ของ user issued by Issuing CA

4. ทดสอบ Login + ปิด Password Login

1
ทดสอบ Smart Card Login

Logout → Login screen → Sign-in options → Smart Card icon → ใส่ PIN → ต้องเข้าได้

2
ปิด Password Login (หลัง confirm กับ Domain Admin)

AD Users and Computers → Properties ของ user → Tab: Account
✔ เลือก "Smart card is required for interactive logon"

3
ทดสอบอีกครั้ง

ต้องไม่สามารถ login ด้วย Password ได้แล้ว

5. กรณี User ลืม PIN หรือถูก Lockout

📌 ก่อนดำเนินการใดๆ

ยืนยันบัตรประจำตัวพนักงาน / ยืนยัน Username ตรงกับเครื่อง — บันทึก Log: ชื่อ IT, ชื่อ User, เวลาเริ่ม, เหตุผล

Decision Tree — อ่าน Error Message ที่หน้า Login

ขึ้น Error: "The PIN has been blocked temporarily..."
กด OK แล้วดูว่ามีอะไรปรากฏ
มี "Unblock smart card"
PUK Path
ใส่ผิดช้าๆ (>5 วิ/ครั้ง)
กด "Unblock smart card"
กรอก PUK จาก Vault
ตั้ง PIN ใหม่ตาม Policy
ทดสอบ Login
เห็นแค่ "Try again later"
TPM Lockout
ใส่ผิดเร็ว (<5 วิ/ครั้ง)
TPM ตีความเป็น Brute-force
ทางเลือก A: รอ 10 นาที–1 ชม.
ให้ TPM คลายล็อกเอง

ทางเลือก B (เร่งด่วน):
Escalate → Destroy + Create ใหม่

6. Destroy + Create (กรณี TPM Lockout หรือ PUK หาย)

⚠ Certificate จะหายทั้งหมด

Destroy VSC = private key ถูกลบถาวร ต้อง Enroll cert ใหม่หลังจาก Create VSC เสร็จ

1
Login ด้วย lcadmin

Login ด้วย local admin account ที่เตรียมไว้ตอน setup

2
Destroy VSC เดิม 
certutil -scinfo   // ดู instance ID
tpmvscmgr.exe destroy /instance ROOT\SMARTCARDREADER\0000
3
Logout แล้ว Login ด้วย Domain User

จำเป็นต้องเป็น domain user เพื่อทำ Create และ Enroll

4
Create VSC ใหม่

ทำตามขั้นตอนที่ 2 ด้านบน — บันทึก PIN/PUK/AdminKey ใหม่ลง Vault

5
Enroll Certificate ใหม่

ทำตามขั้นตอนที่ 3 ด้านบน

6
ปิด Case

User ทดสอบ Login ด้วย PIN ใหม่ต่อหน้า IT — บันทึก Log: เวลาสำเร็จ, วิธีที่ใช้ (PUK หรือ Destroy/Create)
